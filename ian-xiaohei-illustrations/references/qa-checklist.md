# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有“不鸭不咋样”。
- 不鸭不咋样承担核心动作，不只是装饰。
- 不鸭不咋样是白色鸭子、橙嘴橙脚、黑色小眼睛、空呆无语表情。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面怪诞、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色主要用于鸭嘴、鸭脚、主路径或箭头；因为鸭子本身有橙色，其他橙色要克制。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 不鸭不咋样像吉祥物、表情包、玩偶广告、萌宠贴纸或儿童卡通。
- 鸭子只是站在角落里，没有参与核心动作。
- 鸭子太精致、太圆润、太商业，缺少手绘草图感。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太普通，没有荒诞隐喻。
- 和 examples 里的旧案例构图过于相似。

## 迭代方法

- 太普通：让不鸭不咋样成为动作主体，加入一个奇怪但成立的隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 deadpan、blank slightly speechless expression、not cute、not mascot、reluctant but responsible。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 太像旧案例：保留核心意思，换掉主物件和不鸭不咋样的动作。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先觉得“有点怪”，然后很快理解结构。

如果第一眼像教程页，而不是白纸上的怪诞产品草图，就需要继续改。
