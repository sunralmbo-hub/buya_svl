# 不鸭不咋样小漫画正文配图

## 核心定位

为中文文章、帖子、博客、Notion 文档和方法论内容设计 16:9 横版小漫画式正文配图。目标不是做正式流程图、商业插画或精致 PPT 信息图，而是先理解文章里的一个认知锚点，再把它转译成一个“不鸭不咋样”的生活情绪小漫画。

默认视觉 IP 是“不鸭不咋样”：白色长脖子鸭子，橙色扁嘴，黑色小点眼睛，粗黑线轮廓，圆头鼓脸，表情呆傻、无语、可爱、略欠但很有情绪。它可以是一只鸭、两只鸭、几格互动，也可以是一群鸭中的一只；鸭群只是可选方向。

一张图优先表达一个情绪化的认知动作：普通、特别、被看见、被淹没、嘴硬、孤独、闪光、被找到、尴尬、温柔、拥抱、犯困、得意、假装没事。不要把正文做成说明书。

## 先读这些参考

按任务需要读取，不要一次塞满上下文：

- `references/identity-lock.md`：最先读取。锁定不鸭的轮廓、侧脸、嘴型、小点眼、长脖子和禁忌，防止生成普通鸭、黄鸭、真实鸭、3D 玩偶或泛用 mascot。
- `references/style-dna.md`：不鸭小漫画视觉 DNA、分镜、颜色、文字、禁忌。
- `references/buya-ip.md`：不鸭不咋样 IP 的外形、性格、动作、表情和禁忌。
- `references/composition-patterns.md`：不鸭式小漫画构图、单只鸭、两只鸭、鸭群、分镜节奏和原创规则。
- `references/examples/README.md`：按主题选择参考图组，包括鸭群关系、单鸭状态、被找到关系、亲密互动、单鸭身体动作梗。
- `references/prompt-template.md`：单张生图提示词模板。
- `references/qa-checklist.md`：生成后检查和迭代规则。
- `references/reference-policy.md`：参考图使用边界，避免复刻水印、日期、具体原图或未授权构图。

## 工作流

### 1. 先锁角色身份

生成前先读取 `references/identity-lock.md`。如果不鸭角色不像，后面的分镜、文字和构图都不算成功。

优先确保：白色长脖子、大圆头、鼓脸、橙色扁嘴、小黑点眼、粗黑手绘线、简化白色身体。

### 2. 消化正文

先读用户给的正文、链接、Notion 页面、Markdown 文件或截图内容。提炼：

- 核心观点是什么
- 哪个段落有最强的情绪/反差/梗点
- 哪个抽象结构可以转成一个小漫画场景
- 需要单只鸭、两只鸭、几格过程，还是鸭群
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择“认知锚点 + 情绪锚点”重合的位置，例如：核心判断、前后对比、角色状态变化、普通但特别、被看见、被淹没、嘴硬、得意、无语、崩溃、被迫营业。

### 3. 先出配图策略

如果用户只是说“分析怎么配图 / 思考哪些地方需要配图”，先给 shot list。每张图写清楚：

- 放在哪个段落后
- 图的主题
- 核心意思
- 情绪关键词
- 小漫画分镜格式
- 构图类型：单只鸭 / 两只鸭 / 四格 / 鸭群 / 远近切换
- 不鸭在图里做什么
- 建议中文旁白/对白

默认 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。够用就好，避免把正文做成画册。

### 4. 单张生成

如果用户明确要求“生成 / 输出 / 做图 / 帮我生成”，不要停下来等确认；用内置 `image_gen` 每张单独生成。不要把多张图拼在一张里。

每张图只讲一个核心小漫画场景。提示词必须先写 identity lock，再写分镜。必须包含：

- 16:9 横版中文小漫画正文配图
- 纯白页面背景
- 粗黑色手绘漫画分镜边框
- 白色长脖子鸭子、橙色扁嘴、黑色小点眼睛、圆头鼓脸
- 白色手写文本框，深蓝灰或黑色中文手写旁白
- 可选：鸭群重复纹理、浅绿色分镜背景块、特殊鸭、粉色标记、黄色闪光
- 禁止 PPT、正式流程图、复杂架构图、真实 UI、3D 玩偶、毛绒玩具、精致商业吉祥物、普通黄鸭、真实鸭、复刻原图、水印、日期、平台 ID

不要复刻参考图。参考图只用于校准角色外形、线条、颜色和分镜语法，不能直接复用具体构图、文字、道具、日期、水印或场景。

### 5. 检查与迭代

生成后检查 `references/qa-checklist.md`。如果出现以下问题，优先重生成或局部编辑：

- 不像“不鸭不咋样”小漫画
- 不鸭角色不像：脖子短、嘴不扁、眼睛太大、像黄鸭/真实鸭/玩偶/泛用 mascot
- 没有粗黑漫画分镜框
- 橙嘴、小点眼、鼓脸、粗黑线不明显
- 画面太像流程图/PPT
- 中文太多或错字严重
- 左上角出现“流程图/系统架构图/路线图”等类型标题
- 画风太精致、太商业、太 3D、太写实
- 出现水印、日期、平台 ID 或直接复刻参考图

### 6. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 每张图对应的情绪/梗点
- 分镜格式
- 保存路径
- 哪些图最稳，哪些图是可选

不要长篇解释风格理论；让图自己说话。
