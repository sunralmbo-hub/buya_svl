# 不鸭不咋样正文配图

## 核心定位

为中文文章、帖子、博客、Notion 文档和方法论内容设计 16:9 横版正文配图。目标不是做正式流程图、商业插画或精致 PPT 信息图，而是先理解文章里的一个认知锚点，再把它转译成一个“不鸭不咋样”的生活情绪场景。

默认视觉 IP 是“不鸭不咋样”：一只白色长脖子鸭子，橙色扁嘴，黑色小点眼睛，粗黑线轮廓，圆头鼓脸，表情呆傻、无语、可爱、略欠但很有情绪。它不是严肃系统操作员，而是把抽象观点变成生活梗、表情包反应和情绪替身的角色。

一张图优先表达一个情绪化的认知动作：卡住、得意、委屈、无语、发财、晚安、崩溃、装懂、嘴硬、抱住、拖走、被夹住、被问题追着跑。不要把正文做成说明书。

## 先读这些参考

按任务需要读取，不要一次塞满上下文：

- `references/style-dna.md`：不鸭视觉 DNA、线条、颜色、文字、禁忌。
- `references/buya-ip.md`：不鸭不咋样 IP 的外形、性格、动作、表情和禁忌。
- `references/composition-patterns.md`：不鸭式生活梗构图、情绪替身和原创规则。
- `references/prompt-template.md`：单张生图提示词模板。
- `references/qa-checklist.md`：生成后检查和迭代规则。
- `references/reference-policy.md`：参考图使用边界，避免复刻水印、日期、具体原图或未授权构图。
- `assets/references/`：自制不鸭角色校准图，只用于外形、比例、线条和气质参考。

## 工作流

### 1. 消化正文

先读用户给的正文、链接、Notion 页面、Markdown 文件或截图内容。提炼：

- 核心观点是什么
- 哪个段落有最强的情绪/反差/梗点
- 哪个抽象结构可以转成一个生活场景
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择“认知锚点 + 情绪锚点”重合的位置，例如：核心判断、两个断点、输入输出闭环、前后对比、常见坑、角色状态变化、嘴硬、拖延、过载、得意、无语、崩溃、被迫营业。

### 2. 先出配图策略

如果用户只是说“分析怎么配图 / 思考哪些地方需要配图”，先给 shot list。每张图写清楚：

- 放在哪个段落后
- 图的主题
- 核心意思
- 情绪关键词
- 不鸭式场景类型
- 不鸭在图里做什么
- 建议道具
- 建议中文短句

默认 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。够用就好，避免把正文做成画册。

### 3. 单张生成

如果用户明确要求“生成 / 输出 / 做图 / 帮我生成”，不要停下来等确认；用内置 `image_gen` 每张单独生成。不要把多张图拼在一张里。

每张图只讲一个核心场景。提示词必须包含：

- 16:9 横版中文正文配图
- 纯白背景
- 粗黑色手绘线稿
- 白色长脖子鸭子、橙色扁嘴、黑色小点眼睛、圆头鼓脸
- 少量中文手写短句，优先 2-6 个字
- 少量橙色，必要时用粉色/浅绿/浅蓝点缀
- 大量留白
- 不鸭不咋样作为生活情绪和梗的载体
- 禁止 PPT、正式流程图、复杂架构图、真实 UI、3D 玩偶、毛绒玩具、精致商业吉祥物、复刻原图、水印、日历文字

不要复刻过往案例。参考图只用于校准角色外形、线条密度、颜色克制和情绪表达，不能直接复用具体构图、文字、道具、日期、水印或场景。

### 4. 检查与迭代

生成后检查 `references/qa-checklist.md`。如果出现以下问题，优先重生成或局部编辑：

- 不鸭不咋样不像白色长脖子鸭子
- 橙嘴、小点眼、鼓脸、粗黑线不明显
- 画面太像流程图/PPT
- 情绪不清楚，只是在解释结构
- 中文太多或错字严重
- 左上角出现“流程图/系统架构图/路线图”等类型标题
- 画风太精致、太商业、太 3D、太写实
- 出现水印、日期、日历版式或直接复刻参考图
- 背景不是干净白底

### 5. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 每张图对应的情绪/梗点
- 保存路径
- 哪些图最稳，哪些图是可选

不要长篇解释风格理论；让图自己说话。
