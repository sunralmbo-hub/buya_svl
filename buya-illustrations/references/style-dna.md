# 风格 DNA

## 一句话

纯白、粗黑线、白鸭、橙嘴、小点眼、圆头鼓脸、长脖子、表情包感、生活梗、短中文、呆傻可爱、无厘头、情绪直接。

像一张从聊天表情包里长出来的正文配图：它先让人觉得“这鸭好傻”，再让人一下子看懂文章里的状态、判断或反差。

## 必须

- 16:9 横版正文配图。
- 纯白背景：不要米色、暖灰、纸张纹理、渐变、阴影、噪点、复古纸感。
- 黑色手绘线稿为主：线条要比普通草图更粗、更圆、更稳定；不要矢量化，不要过细。
- 固定角色是“不鸭不咋样”：白色长脖子鸭子、橙色扁嘴、黑色小点眼睛、圆头鼓脸、简化白色身体。
- 角色必须承担生活情绪或梗点：卡住、无语、得意、委屈、嘴硬、拖延、发财、装懂、崩溃、被迫营业。
- 大量留白：主体占画面约 35%-55%，至少 40% 空白，最好有一整块安静区域。
- 少量中文手写短句：最多 1-3 处，每处尽量 2-6 个字。
- 一张图只讲一个核心情绪场景、生活梗或隐喻动作。
- 结构要自然藏在场景里，不要在图上写结构类型名称。

## 颜色

- 黑色：主体轮廓、鸭子的眼睛、框线、短句、主要物件。
- 白色：不鸭不咋样的身体。白鸭必须靠粗黑轮廓从白底中分离出来。
- 橙色：鸭嘴、鸭脚、少量高光、动作线、关键道具。橙色是识别色，不能铺满。
- 粉色：腮红、害羞、尴尬、爱心、轻微情绪。
- 浅绿/浅蓝：衣服、书本、背景块、辅助道具。只能少量点缀。
- 红色：只用于强提醒、惊讶或结果，不要常用。

颜色要克制。主画面必须仍然是黑白橙，不要变成彩色儿童插画。

## 绝对不要

- 不要商业插画。
- 不要 PPT 信息图。
- 不要正式流程图。
- 不要课程课件。
- 不要复杂架构图。
- 不要精致扁平插画。
- 不要科技感 UI。
- 不要真实 App 截图。
- 不要 3D、毛绒玩具、潮玩公仔、真实鸭子或拟真动物。
- 不要复杂背景、渐变、阴影、纸纹、米色底、噪点。
- 不要把每个节点都解释清楚。
- 不要左上角写“Workflow 流程图 / 系统架构图 / 常见坑 / 路线图”等类型标题。
- 不要出现水印、日历日期、平台 ID 或参考图里的原始文字。

## 审美方向

要像生活表情包、头像、壁纸小图和短梗插画：呆傻、可爱、无厘头、情绪明确、画面简单。

“不鸭不咋样”的气质不是严肃工具人，而是一个把抽象概念变成生活情绪反应的白鸭替身。
